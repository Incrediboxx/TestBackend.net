Записка к тестовому заданию
Некоторые данные касательно Акций и облигаций мог воспринять не так, из-за чего некоторые поля объектов могли остаться пустыми
Заполнение данных происходит в AppsContext в методе InitDB
Из за выбранного типа бд и нехватки опыта с ним пришлось продублировать методы в контроллерах. При работе в обычной бд можно было бы вернуть DataSet с нужными полями и присвоить их экземпляру класса, без обращений к _context

- получение перечня всех эмитентов;
https://localhost:44385/api/company

- получение перечня всех активов эмитента по его id
https://localhost:44385/api/company/byId?id=1

- получение полной информации об активе по id;
https://localhost:44385/api/asset/byId?id=1
- получение полной информации об активе по id;
https://localhost:44385/api/asset/byIsin?isin=RU000A1013J4

- поиск активов
Имена параметров: namePart, ticketPart,isinPart, assetClass
Пример вызова
https://localhost:44385/api/asset/byPart?isinPart=RU000A
